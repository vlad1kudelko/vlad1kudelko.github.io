# Кем становится инженер рядом с автономными системами: AI-агенты и новые роли

AI-агенты меняют распределение работы: часть действий выполняет модель, часть — tools, часть остаётся за человеком. Возникают новые инженерные роли вокруг постановки задач, контроля качества и безопасности.

## Agent operator

Оператор агента следит за задачами, очередями, ошибками и подтверждениями. Это похоже на эксплуатацию автоматизации: агент работает сам, но человек понимает, когда вмешаться.

## Eval engineer

Без evals агент быстро становится непредсказуемым. Нужны наборы задач, regression tests, метрики качества, сравнение моделей и анализ провалов. Eval engineer превращает субъективное «стало хуже» в измеримый процесс.

## Tool designer

Агент силён настолько, насколько аккуратно спроектированы tools. Инженер описывает контракты, ограничения, idempotency, права и формат ошибок. Хороший tool не даёт модели случайно выполнить опасное действие.

## Security reviewer

AI-агенты работают с файлами, токенами, репозиториями и внешними API. Нужны проверки prompt injection, data leakage, permissions и audit. Security reviewer становится обязательной ролью для agentic-систем.

## Итог

AI-агенты не отменяют инженеров. Они создают спрос на людей, которые умеют проектировать, ограничивать, проверять и эксплуатировать автономные системы.
